Programando eh Shell Script no Debian

Variáveis

As variáveis guardam informações sobre o sistema e sobre as preferências do usuário. Elas podem ser divididas em duas categorias: Variáveis de Ambiente São conhecidas como variáveis globais porque podem ser lidas por todos os processos que o shell iniciar. Por convenção, elas são declaradas em caixa alta. O bash mantém algumas variáveis ambientais importantes:  PATH: Seu conteúdo mantém uma lista de diretórios que o shell procura pelos executáveis chamados na linha de comando.  HOME: Especifica o diretório HOME do usuário corrente.  USER: Especifica o login do usuário corrente.  TERM: Declara o tipo de terminal utilizado.

Variáveis de Shell

São também chamadas de variáveis locais, pois são válidas somente no shell em execução. Os processos que executam a partir do shell não tem acesso a elas. Elas são por convenção declaradas em caixa baixa. Como o shell não faz distinção entre as variáveis declaradas em caixa alta ou baixa, todas as variáveis são consideradas variáveis de shell até que sejam exportadas com o comando export. Aliases Outra facilidade importante do shell é a possibilidade de se criar apelidos ou atalhos para os comandos. Ele pode ser utilizado para simplificar comandos com muitas opções ou executar diversos processos em sequência. Funções O bash possibilita definir funções do usuário. Uma função pode chamar uma série de comandos de forma mais complexa que o alias. As funções tem a seguinte sintaxe: # [ function ] NOME () { COMANDOS; } A palavra reservada function é opcional. O NOME é como a nova função será chamada. Os comandos devem estar compreendidos entre chaves.

Arquivos de Configuração do Bash

O bash possibilita que as funções, variáveis e apelidos possam ser gravados em alguns arquivos para que possam ser carregadas novamente quando o sistema for iniciado, ou quando uma nova execução do bash for feita. Alguns arquivos lidos pelo bash: /etc/profile Arquivo global de configuração de variáveis executado durante o processo de autenticação. /etc/bashrc Arquivo global de configuração de variáveis executado toda vez que o bash é carregado. ~/.bash_logout É executado durante o processo de logout Alterando e Escrevendo Scripts de Shell Para criar scripts é preciso enxergar o sitema como uma coleção de ferramentas que podem interagir entre si para formar pequenos programas que irão auxiliar o dia-a-dia de trabalho e manutenção do sitema. Estes pequenos programas são escritos em texto puro em um editor de textos comum e não precisam ser compilados. O shell é capaz de ler este programa como uma lista de comandos. Existem diversas linguagens de programação que interpretam os comandos de um arquivo texto. Desta forma, a primeira linha de um script deve indicar qual será o interpretador utilizado para executar os comandos

Permissões de execução

Os scripts são executados sob as permissões do usuário que os carrega. Mas existem algumas situações especiais onde pode ser necessário executar ações em nome de outro usuário. Para elas é necessário alterar os bits SUID SGID de um processo. Se estes bits estiverem habilitados, o processo será executado com as permissões do usuário que é o seu dono.

Variáveis Especiais

Uma característica importante dos scripts de shell é a possibilidade de configuração da execução com sucesso de um determinado comando. Ela é chamada de variável de retorno. A variável de retorno é preenchida toda vez que um comando é executado. Se o valor retornado for 0, o comando foi executado com sucesso. Se o valor for diferente de 0, algum problema foi encontrado. Se for utilizada em scripts, o seu valor precisa ser verificado imediatamente após a execução do comando desejado.

O comando test

O comando test é uma grande ferramenta para testar arquivos, permissões, textos, números, etc. Ele pode testar diversas condições através de expressões. Ele fornece o resultado do teste através da variável de retorno.

Referências dos Comandos Bash para Programação Shell

O shell possui estruturas para testar condições e executar determinadas sequências de programas várias vezes (loop). Controle de fluxo são comandos que testam algumas alternativas, e de acordo com o resultado executam comandos. A lógica do comando if é “se isso for verdadeiro, faça isso, se não, faça aquilo”

A Condicional case

O comando case é também utilizado para controle de fluxo, mas permite que uma série de valores e expressões regulares seja testada e os respectivos comandos sejam executados

Uso de Loops utilizando o for

O comando for permite que laços sejam feitos para que um ou mais comandos sejam executados até que uma determinada variável percorra todos os valores de uma dada lista

Uso de Loops Utilizando o While

O while testa continuamente uma expressão, até que ela se torne falsa.
